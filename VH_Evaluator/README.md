# VH_Evaluator

**Visual Hallucination Evaluator** - A tool for detecting over-optimization hallucinations in AI-generated images.

## Overview

VH_Evaluator is designed to evaluate visual hallucination phenomena in images generated by diffusion models and other generative AI systems. It specifically detects:

1. **Grid Pattern Artifacts** - Regular grid textures and repetitive patterns caused by model architecture
2. **Texture Over-Enhancement** - Unnatural texture details from over-optimization
3. **Detail Sharpness Issues** - Over-sharpened edges and high-frequency noise
4. **Irrelevant Details** - Excessive decorative content unrelated to the prompt

## Architecture

The evaluation system uses a Vision-Language Model (VLM) to analyze images and provide structured assessments. The workflow:

1. Load target images from a folder
2. Send images to a VLM server (e.g., Qwen2.5-VL) along with reference examples
3. VLM evaluates each image and returns structured JSON scores
4. Generate summary statistics and reports

## Quick Start

### 1. Start the VLM Server

```bash
# Start the evaluation server (requires GPU)
bash start_server.sh
```

This launches a vLLM server with Qwen2.5-VL-72B-Instruct model.

### 2. Run Evaluation

```bash
# Evaluate images in a folder
bash eval.sh /path/to/images /path/to/output.jsonl
```

Or use Python directly:

```bash
python vl_hallucination_client.py \
  --api_url http://localhost:8000 \
  --folder /path/to/images \
  --output_path ./results.jsonl \
  --limit 400 \
  --max_workers 4
```

## Output Format

### Per-Image Results (JSONL)

Each line contains evaluation for one image:

```json
{
  "image_name": "example.jpg",
  "evaluation": {
    "image": {
      "has_over_optimization": true,
      "over_optimization_level": 3,
      "detail_sharpness_score": 2,
      "irrelevant_details_score": 3,
      "grid_pattern_score": 4,
      "overall_hallucination_score": 3,
      "confidence": 0.85,
      "detailed_analysis": "..."
    }
  }
}
```

### Summary Report (JSON)

Aggregated statistics across all images:

```json
{
  "evaluation_summary": {
    "total_images": 100,
    "successful_images": 98,
    "success_rate": 0.98
  },
  "hallucination_scores": {
    "avg_score": 2.5,
    "median_score": 2.0,
    "std_dev": 0.8
  },
  "detailed_metrics": {
    "detail_sharpness_avg": 2.1,
    "irrelevant_details_avg": 2.3,
    "grid_pattern_avg": 3.1
  }
}
```

## Scoring Criteria

All scores range from **0-5**:

| Score | Level | Description |
|-------|-------|-------------|
| 0 | None | No detectable issues |
| 1 | Minimal | Very subtle, barely noticeable |
| 2 | Mild | Noticeable upon close inspection |
| 3 | Moderate | Clearly visible artifacts |
| 4 | Severe | Significant quality degradation |
| 5 | Extreme | Severe hallucination artifacts |

## Low-Level Metrics

For traditional image quality metrics without VLM, use:

```bash
python low_level_metric.py /path/to/images /path/to/output_dir
```

This computes:
- **Sharpness** - Laplacian variance
- **High-Frequency Energy** - High-pass filter response
- **Edge Artifact** - Contrast in edge regions
- **Noise Level** - Noise in smooth regions

## Files

| File | Description |
|------|-------------|
| `vl_hallucination_client.py` | Main evaluation client |
| `low_level_metric.py` | Traditional image quality metrics |

| `eval.sh` | Evaluation convenience script |
| `example/` | Reference images for few-shot evaluation |
